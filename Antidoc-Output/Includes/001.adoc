== TSC File IO.lvlib

*Responsibility:*
+++This set of VIs helps to access configuration-style VIs and helps to adapt to changes when using the current VI's path primitive for a VI that gets packaged into a ppl. Building a ppl adds its namespace and makes the relative path building get messed up without stripping out that namespace.+++


*Version:* 1.0.0.0

|image:TSC_File_IO.lvlib_Read_Section___Path.vi.png[TSC File IO.lvlib:Read Section - Path.vi]
|+++This VI will read a section/key configuration file and return:+++
+++<b>Path</b>+++

+++<b>Note:</b> The VI will resolve relative paths that will be relative to the input ini file path and the symbolic paths. This gives flexibility to defining key/value pairs for absolute and relative paths.+++

+++<appdir> : use the input application directory or the application directory primative (if blank)+++
+++<temp> : the OS temporary directory+++
+++<userdoc> : logged in user's documents folder (Windows)+++
+++<userdesktop> : logged in user's desktop folder (Windows)+++
+++<programdata> : ProgramData folder (Windows)+++
+++<bootdir> : the OS install drive (Windows)+++
+++<programfiles> : the installation directory of applications (Windows)+++
+++..\ : nomenclature of relative path to the ini file directory+++

+++Inputs:+++
+++-Path [path] - a path to the file+++
+++-section [string] - the section string in the configuration file+++
+++-key [string] - the key to be read in the configuration file+++
+++-error in+++

+++Outputs:+++
+++-value [Path] - the output data read+++
